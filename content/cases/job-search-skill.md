# Job Search Materials Skill

场景：个人工作流 / 资料自动化 / Skill / 知识库

## 背景

求职资料往往分散在历史简历、项目文档、岗位 JD、面试反馈和聊天建议中。如果每次投递都重新整理，会消耗大量时间，也难以持续迭代。

## 问题

- 原始资料分散。
- 简历版本、技能矩阵、项目故事和面试话术之间缺少连接。
- 岗位反馈没有回写到资料系统。
- 求职材料容易变成一次性文档，而不是可复用资产。

## 行动

- 设计“原始资料 -> 技能矩阵 -> 项目故事 -> 简历版本 -> 面试话术 -> 投递反馈”的流程。
- 建立 JobSearch-Kit 目录，沉淀求职资料、岗位跟踪表和可复用 Skill。
- 将简历拆成 DevOps、技术支持、AI 自动化等不同版本。
- 用项目故事库支撑简历、面试和个人主页。

## 结果

- 形成可复用的求职资料生产系统。
- 能围绕不同岗位快速定制简历和表达。
- 让求职、内容输出和个人网站共享同一套素材。

## 能力证明

这个案例证明我能把复杂的个人流程标准化、工具化，并持续迭代。
